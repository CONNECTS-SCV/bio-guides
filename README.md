브라우저만 있으면 시작할 수 있는 한국어 실습 가이드 모음입니다. 각 과정은 **본문(마크다운) + 실습 노트북 + 실제 실행 결과**로 이뤄져 있고, 노트북은 **여러분이 도구를 직접 돌려 자기 데이터를 만들고** 그 결과로 다음 단계를 이어가도록 짜여 있습니다.

| 과정 | 폴더 | 무엇을 하나 |
|------|------|-------------|
| **BoltzGen — AI 바인더 설계** | [`boltzgen/advanced/`](boltzgen/advanced/00_README.md) | 타깃에 결합하는 단백질·펩타이드·항체·나노바디·소분자·핵산 바인더를 생성형 AI로 설계하고, 메트릭으로 선별합니다. |
| **항체 데이터베이스 분석** | [`ab db/antibody_db_guide/`](ab%20db/antibody_db_guide/00_README.md) | SAbDab·OAS 등 항체 DB를 다루고, numbering·humanness·구조·인터페이스·개발성을 분석합니다. |

## 실습하는 법

각 챕터 폴더 안의 `.ipynb`를 GitHub에서 열고 **Open in Colab** 으로 실행하면 됩니다 — **고칠 값은 없습니다.** 첫 셀이 이 저장소를 클론하고 해당 챕터 폴더로 자동 이동한 뒤, 필요한 라이브러리를 그 자리에서 설치합니다.

로컬 주피터에서 챕터 폴더 안의 노트북을 열어도 클론 없이 그대로 동작합니다. (fork 했다면 노트북 첫 셀의 `REPO_URL`을 본인 주소로 바꾸세요.)

### 노트북이 도는 방식

각 랩은 세 단계로 진행됩니다.

1. **직접 실행** — 도구를 실제로 돌려 산출물을 챕터 폴더의 `my_run/`에 만듭니다.
2. **내 결과 확인** — 방금 만든 결과를 읽어 표·그래프를 그립니다.
3. **레퍼런스 대조** — 저장소에 커밋된 `data/`(실제 실행으로 만든 대조군)와 비교합니다.

실행을 건너뛰거나 중간에 실패해도 실습이 끊기지 않습니다 — `my_run/`이 없으면 자동으로 레퍼런스 결과로 이어집니다(노트북이 어느 쪽을 쓰는지 출력해 줍니다).

`my_run/`은 `.gitignore` 처리되어 있어, 여러분의 실행 결과가 저장소를 더럽히지 않습니다.

## 저장소 구조

```
boltzgen/advanced/        BoltzGen 과정 (챕터 01~12 + 노트북 10)
ab db/antibody_db_guide/  항체 DB 분석 과정 (챕터 01~10 + 노트북)
humanization/             항체 인간화 과정
  humanization_guide/       챕터 01~11 + 노트북
  _source/                  분할 전 원본 문서(아카이브)
```

각 과정 폴더에는 `build_combined.py`(챕터 → 단일 `combined.md`)와 `gen_notebooks.py`(노트북 생성기)가 있습니다. **노트북은 손으로 고치지 말고 생성기를 고친 뒤 재생성**하세요.

## 수치에 대하여

본문의 모든 명령·수치·그래프는 해당 도구를 **실제로 실행해 얻은 결과**입니다. 각 과정의 부록에 재현 환경(도구 버전·실행 시간)을 정리해 두었습니다.
